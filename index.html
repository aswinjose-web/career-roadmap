Phase 1 Build Project

CLI Code Review Assistant

Build a command-line tool that accepts a file path or Git diff and returns a structured code review using the Anthropic API.

  • Accept input from stdin or a file argument
  • Use a well-crafted system prompt defining a senior reviewer persona
  • Return structured output: severity, category, line reference, suggestion
  • Handle token limit gracefully (chunk large files)
  • Bonus: add a --team-standards flag to inject your team's style guide

Applied building

Building with AI — RAG & Data Systems

⏱ Weeks 11–18 · 5–7 hrs/week

RAG (Retrieval-Augmented Generation) is the most important architecture in enterprise AI right now. It's how you connect LLMs to your company's own data — docs, code, tickets, wikis — without retraining a model. Understanding RAG end-to-end makes you valuable in any technical AI conversation. You'll learn embeddings, vector databases, chunking strategies, and retrieval ranking.

Skills you'll build
Text embeddings Vector databases (Chroma, Qdrant) Semantic similarity search Chunking strategies Document loaders Retrieval reranking LlamaIndex / LangChain Hybrid search (BM25 + semantic) RAG evaluation (RAGAS)

Phase 2 Build Project

Internal Team Knowledge Base Q&A

Build a RAG system that answers questions over your team's actual documentation — READMEs, Confluence pages, Notion docs, or runbooks.

  • Load documents from a folder (PDFs, Markdown, plain text)
  • Chunk intelligently (by heading, not arbitrary token count)
  • Store embeddings in Chroma locally
  • Answer questions with source citations ("Found in: auth-service/README.md")
  • Evaluate: create 10 test questions, measure answer quality with RAGAS
  • Bonus: add a Slack slash command so your team can use it

Advanced building

AI Agents & Multi-Step Orchestration

⏱ Weeks 19–26 · 5–7 hrs/week

Agents let LLMs take actions — calling APIs, browsing the web, writing and running code, searching databases — and chain multiple steps together autonomously. This is the frontier of applied AI engineering. As a manager, knowing agent failure modes and how to architect reliable agentic systems makes you irreplaceable in technical planning conversations.

Skills you'll build
Tool use / function calling ReAct reasoning pattern Multi-agent architectures LangGraph stateful agents Anthropic tool use API MCP (Model Context Protocol) Agent debugging & tracing Guardrails & safety Human-in-the-loop patterns

Phase 3 Build Project

Automated Sprint Retrospective Agent

Build an agent that reads your team's completed sprint (Jira/Linear/GitHub issues) and generates a retrospective report automatically.

  • Tool 1: Query your project management API for closed issues in date range
  • Tool 2: Read PR descriptions and commit messages from GitHub
  • Tool 3: Look up any linked Slack threads (or Notion docs)
  • Agent synthesizes: what went well, blockers, patterns, action items
  • Add human-in-the-loop: agent pauses and asks for clarification on ambiguous items
  • Bonus: post the report to Slack automatically with an approval step

Phase 5 Capstone

Ship something your team actually uses

By this point, build something real that solves a real problem for your team. The goal isn't to showcase AI — it's to solve a real problem. Ideas:

  • An internal AI assistant over your company's documentation + Jira + Slack
  • An automated PR description generator that follows your team's conventions
  • A fine-tuned model for your domain (legal, fintech, healthcare) that your team relies on
  • A monitoring dashboard that alerts on AI output quality degradation in production

Your edge isn't just knowing AI — it's knowing how to build with it and lead teams through it.

Revisit this guide every 3 months. The tooling changes fast; the fundamentals don't.
